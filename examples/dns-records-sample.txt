; =============================================================================
; Sample DNS Records — example.com
; (Sanitized — all values are placeholders)
;
; Shows what a complete, hardened set of DNS records for a web domain
; looks like in zone-file format. Use as a visual reference when
; configuring records in a DNS provider's UI.
; =============================================================================

; --- A and AAAA records (the actual site) ---
@           IN  A     203.0.113.10
@           IN  AAAA  2001:db8::10
www         IN  A     203.0.113.10
www         IN  AAAA  2001:db8::10

; --- MX records (mail routing) ---
@           IN  MX    1  aspmx.l.google.com.
@           IN  MX    5  alt1.aspmx.l.google.com.
@           IN  MX    5  alt2.aspmx.l.google.com.

; --- CAA records (which CA can issue certificates) ---
@           IN  CAA   0 issue "letsencrypt.org"
@           IN  CAA   0 issuewild "letsencrypt.org"
@           IN  CAA   0 iodef "mailto:security@example.com"

; --- SPF (which servers can send mail for this domain) ---
@           IN  TXT   "v=spf1 include:_spf.google.com ~all"

; --- DKIM (cryptographic signature for outbound mail) ---
; The actual p= value will be hundreds of characters long. The string
; below is a clearly-marked placeholder.
google._domainkey  IN  TXT  "v=DKIM1; k=rsa; p=PLACEHOLDER_PUBLIC_KEY_VALUE_FROM_MAIL_PROVIDER"

; --- DMARC (what receivers do with failing mail) ---
; Start with p=none and a reporting address. Progress to p=quarantine,
; then p=reject after 30 to 60 days of clean reports.
_dmarc      IN  TXT   "v=DMARC1; p=none; rua=mailto:dmarc-reports@example.com; ruf=mailto:dmarc-forensic@example.com; fo=1"

; --- DNSSEC ---
; DNSSEC is not represented as a zone file record. It is enabled at the
; DNS host (Cloudflare, Route 53, BIND auto-signing). The DS record
; lives at the parent zone (the registrar), NOT in this zone file.
;
; Example DS record values published at the registrar:
;   12345 13 2 ABCDEF1234567890PLACEHOLDER_DIGEST_VALUE
;
; The "12345" is the key tag, "13" is the algorithm (ECDSA Curve P-256),
; "2" is the digest type (SHA-256). These values are generated by the
; DNS host when DNSSEC is enabled.

; --- Notes ---
; This is a zone-file representation. In practice, most operators use
; a DNS provider's web UI (Cloudflare, Route 53, DigitalOcean DNS) which
; abstracts away the syntax. The records above are equivalent regardless
; of how they are entered.
;
; TTL values are not shown above and inherit from the zone default.
; Reasonable defaults: 300 (5 minutes) for records that may change,
; 3600 (1 hour) for stable records, 86400 (1 day) for very stable
; records like MX.
